2

Data Collection & Preprocessing

Week 2 | Data Acquisition Phase
Member 1 Data Collection
  • Collect SBI, HDFC, ICICI stock data (minimum 3 years)
  • Use Yahoo Finance API or yfinance library
  • Download OHLCV data (Open, High, Low, Close, Volume)
  • Ensure consistent date ranges across all stocks
  • Save raw data in CSV format
Member 2 Data Exploration
  • Perform exploratory data analysis (EDA)
  • Create visualization of stock price trends
  • Check for missing values and outliers
  • Analyze data distribution and statistics
  • Generate correlation matrices between stocks
Member 3 Data Cleaning
  • Handle missing values (forward fill/interpolation)
  • Remove duplicates and inconsistencies
  • Filter data for trading days only
  • Validate data integrity across all three banks
  • Document all cleaning decisions
Member 4 Data Preprocessing
  • Normalize/scale data using MinMaxScaler (0-1 range)
  • Create train-test split (80:20 ratio)
  • Prepare sequences for LSTM (windowing)
  • Calculate daily returns for risk analysis
  • Save preprocessed data for modeling

📦 Week 2 Deliverables

  • Clean dataset for SBI, HDFC, ICICI (CSV files)
  • EDA report with visualizations
  • Preprocessed data ready for LSTM
  • Data documentation (source, period, features)

💡 Pro Tips

Use at least 3 years of historical data for better model training. Consider using data from 2021-2024 to include recent market volatility. Document all preprocessing steps for reproducibility.

3

LSTM Model Development

Week 3 | Model Building Phase
Member 1 Data Pipeline
  • Create data generators for LSTM input
  • Implement sliding window approach (e.g., 60-day window)
  • Reshape data into (samples, timesteps, features)
  • Set up batch processing
  • Validate input shapes for model compatibility
Member 2 LSTM Architecture
  • Design LSTM model architecture (2-3 layers)
  • Set LSTM units per layer (e.g., 128, 64, 32)
  • Add dropout layers (0.2-0.3) for regularization
  • Configure output layer (Dense layer with 1 unit)
  • Implement model using Keras Sequential API
Member 3 Hyperparameter Tuning
  • Select optimizer (Adam recommended)
  • Set learning rate (start with 0.001)
  • Define loss function (MSE for regression)
  • Determine batch size (32, 64, or 128)
  • Configure epochs (50-100) with early stopping
Member 4 Model Training Setup
  • Implement model compilation
  • Set up callbacks (ModelCheckpoint, EarlyStopping)
  • Configure validation split or validation set
  • Prepare training monitoring (TensorBoard/logs)
  • Document model architecture and parameters

📦 Week 3 Deliverables

  • Complete LSTM model architecture code
  • Model architecture diagram
  • Hyperparameter configuration document
  • Training pipeline ready for execution

💡 Pro Tips

Start with a simple 2-layer LSTM and gradually add complexity. Use return_sequences=True for stacked LSTM layers. Save model checkpoints to avoid losing trained weights.

5

Anomaly Detection Implementation

Week 5 | Anomaly Detection Phase
Member 1 LSTM Autoencoder Design
  • Design encoder-decoder architecture
  • Implement encoder (LSTM + RepeatVector)
  • Implement decoder (LSTM + TimeDistributed)
  • Set bottleneck layer size (e.g., 128 units)
  • Compile autoencoder with MSE loss
Member 2 Autoencoder Training
  • Train autoencoder on normal stock data
  • Use 80:20 split for train-test
  • Monitor reconstruction loss
  • Save trained autoencoder models (all banks)
  • Generate reconstruction error plots
Member 3 Anomaly Detection Logic
  • Calculate reconstruction errors on test data
  • Set threshold using statistical methods (mean + 2*SD)
  • Identify anomalous points above threshold
  • Mark anomaly dates and severity
  • Create anomaly visualization (red dots on price chart)
Member 4 Anomaly Analysis
  • Compare anomaly patterns across SBI, HDFC, ICICI
  • Correlate anomalies with real events (COVID, elections, etc.)
  • Create test loss vs threshold plots
  • Generate anomaly detection report
  • Document detected anomalies with explanations

📦 Week 5 Deliverables

  • LSTM autoencoder models for all banks
  • Anomaly detection results with marked dates
  • Visualization of anomalies on price charts
  • Anomaly analysis report with event correlation

💡 Pro Tips

Reference paper found anomalies in BBRI but not BBCA/BMRI. Document whether your findings align or differ, and explain why. Consider using multiple threshold methods (percentile, Z-score).

🎓 Overall Project Success Tips

📚 Technical Tips

  • Use GPU/Colab for faster training
  • Save checkpoints regularly
  • Version control your code
  • Test on small data first

👥 Team Collaboration

  • Daily standups (15 min)
  • Weekly progress reviews
  • Use Slack/Discord for comms
  • Share code on GitHub

📊 Quality Metrics

  • Aim for MSE < 10,000
  • MAPE should be < 5%
  • Train multiple times, average results
  • Compare with baseline models

⚠️ Common Pitfalls

  • Don't overfit (use dropout)
  • Scale data properly
  • Handle missing data carefully
  • Document everything!

📄 Reference Paper Insights

Your reference paper analyzed Indonesian banks (BBRI, BBCA, BMRI). Here's what they found:

  • Best Performer: BBCA (no anomalies, lowest risk 1.18%)
  • Prediction Accuracy: BBRI had MAE 64.13, MSE 7341.28
  • Risk Values: BBRI (1.57%), BMRI (1.51%), BBCA (1.18%)
  • Anomaly Detection: Only BBRI showed anomalies in their test period
  • Model: Used 4-layer LSTM autoencoder (128 units each) for anomaly detection

Your Goal: Replicate this methodology for Indian banks (SBI, HDFC, ICICI) and compare results. Indian banking sector may show different patterns due to market dynamics, regulations, and economic conditions.
